Níveis de Testes.

qui vai uma descrição básica sobre os fundamentos dos níveis de testes, será editádo futuramente.

Testes de Unidade

O objetivos dos testes de unidade são, encontrar falhas de funcionamento dentro de uma pequena parte do sistema independentemente do todo. Este teste é realizado pelo programador durante e após a codificação e o alvo dos testes são as sub-rotinas, métodos ou seja as menores unidades do sistema. Geralmente é automatizado, através de ferramentas como Junit, PHPUnit entre outros, precisa sempre estar atualizado e coerente com as regras atuais do sistema.

Teste de Integração

O teste de integração tem a função de validar á comunicação entre componentes do sistema, também é realizado pelo programador, o alvo são as funcionalidades que envolvem a integração de componentes, geralmente os tipos de falhas encontradas são de transmissão de dados,exemplo mais comum é um componente A invoca um método de um componente B, e espera um valor inteiro, porém, vem um valor decimal, causando falha no componente A.

Geralmente é automatizado, através de ferramentas e podem ser realizados antes do sistema estar concluído, á medida em que os componentes vão ficando prontos.
